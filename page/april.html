Blog Kegiatan Magang Bulan Ketiga

Sebagai mahasiswa magang di PT Ganeshcom Mitra Solusi Digital magang di bulan ketiga ini membuat saya mengetahui berberapa hal yang penting dalam membuat sebuah project yang baik.

#INTERNATGANESHCOMSTUDIO

Minggu 1 - Analisis Flow Untuk Website CMS

Pada awal minggu di bulan April, kami satu tim mahasiswa magang memulai dengan fokus pada pengembangan Website Company Profile Ganeshcom. Task saya di hari pertama adalah melakukan konfigurasi slicing yang telah dibuat ke dalam framework Laravel, dengan menerapkan konsep layerize sesui dengan sturtur code pada perusahaan agar menerapkan clean code pada project yang dibuat. Penggunaan konsep layerize untuk mempermudah pengembangan dan pemeliharaan kode kedepannya

Di hari selanjutnya, saya terlibat dalam analisis komponen-komponen Website halaman admin (CMS Ganeshcom Company profile) untuk menerapkan fungsi CRUD (Create, Read, Update, Delete) pada setiap bagian section content pada website. Kami satu tim melakukan analisis terhadap setiap bagian, termasuk Header, Hero, Trusted, Service, Expertise, Project, Testimoni, Client, Contact, dan Footer Section, untuk memastikan bahwa fungsionalitas websie admin page dapat diintegrasikan dengan lancar.

Selanjutnya saya juga tetap fokus pada perbaikan bug yang muncul, terutama terkait proses logout admin. Selain itu, dihari berikutnya kami satu tim dalam project juga mulai merancang struktur database secara menyeluruh dari project Website Company Profile Ganehscom yang dibantu oleh mentor magang di Ganeshcom, hal tersebut dilakukan sebagai langkah awal sebelum memulai implementasi (CRUD pada admin page). Proses ini memerlukan analisis untuk memastikan bahwa database dapat mendukung semua fitur yang akan kami implementasikan. Dalam perancangan database selalu ada komunikasi antara tim dan mentor supaya dapat meminimalisir terjadinya miskomunikasi satu sama lain.

Terakhir, saya dan tim melanjutkan dengan analisis database keseluruhan dan proses slicing Interface Admin Page. Kami membagi tugas dalam mengimplementasikan menu Dashboard, Seo Management, Config, dan User Management, dengan mempertimbangkan penggunaan konsep layerize. Dengan berbagai langkah yang dilakukan itu untuk memastikan bahwa pengembangan Website Company Profile Ganeshcom berjalan lancar dan sesuai dengan rencana yang telah dibuat.

Minggu 2 - Membuat Fitur Baru Pada Project Ganeshcom Company Profil

Pada minggu kedua bulan April, fokus proyek pada pengembangan fitur-fitur penting dalam Website Ganeshcom Company Profile. Task saya pada minggu kedua ini adalah membuat Fitur User Management yang meliputi fungsi Create, Read, Update, Delete, dan Block User. Saya menyusun dan mengimplementasikan fitur ini dengan teliti, dengan tujuan untuk memberikan pengalaman pengguna yang lancar dan efisien. Selanjutnya, saya melanjutkan dengan pembuatan Fitur Configuration, yang mencakup fungsi untuk read dan mengupdate konfigurasi website.

Setelah menyelesaikan task-task yang tersedia dalam pengembangan tersebut, tim Ganeshcom melakukan buka bersama pada tanggal 8 April 2024 setelah jam kantor usai. Acara ini diselenggarakan untuk mempererat tali persaudaraan dan meningkatkan toleransi antar sesama staff dan mahasiswa magang di PT Ganeshcom Mitra Solusi Digital. Saya melihat acara ini bukan hanya sebagai kesempatan untuk bersenang-senang, tetapi juga sebagai momen penting untuk membangun hubungan yang lebih erat dan solid di antara staff dan mahasiswa magang di PT Ganeshcom Mitra Solusi Digital.

Buka bersama ini memberikan kita semua waktu untuk berbagi cerita, pengalaman, dan kebahagiaan bersama. Ini adalah kesempatan untuk meningkatkan kolaborasi antara kami sebagai tim. Semangat kebersamaan yang tercipta dari acara ini menjadi modal berharga dalam pengembangan proyek-proyek kedepannya.

#INTERNATGANESHCOMSTUDIO

Minggu 3 - Belajar Pentingnya Clean Code dan Kenyamanan Pengguna (UX)

Pada minggu ketiga bulan April, task saya adalah pada pengembangan fitur-fitur penting dalam Website Company Profile Ganeshcom. Pada hari pertama, saya mulai dengan melakukan revisi code dari mentor terkait fitur User Management dan Web Configuration. Langkah ini melibatkan pembaruan dan penyesuaian kode berdasarkan masukan yang diberikan oleh mentor, dengan tujuan untuk meningkatkan kualitas fitur tersebut.

Selanjutnya, saya melanjutkan dengan melakukan refactor code dan menerapkan konsep layerize dari mentor terkait fitur User Management dan Web Configuration. Proses refactor ini bertujuan untuk meningkatkan struktur dan menerapkan clean code, serta menerapkan pendekatan yang lebih terstruktur dalam pengembangan fitur-fitur tersebut. Dengan menerapkan konsep layerize, untuk dapat mempermudah proses pengembangan, pemeliharaan, dan penambahan fitur baru kedepannya.

Pada hari-hari berikutnya, saya kembali melakukan revisi code dari mentor terkait fitur Web Configuration. Selain itu, saya juga mengerjakan task untuk menambahkan Image Skeleton pada fitur Edit Web Configuration. Fitur ini dilakukan untuk memperbaiki tampilan dan pengalaman pengguna saat melakukan pengaturan pada website. Dengan penambahan Image Skeleton, pengguna dapat melihat indikasi visual yang jelas saat proses pengaturan sedang berlangsung, sehingga meningkatkan kejelasan dan kenyamanan pengguna dalam menggunakan fitur tersebut hal ini dilakukan untuk meningkatkan UX(User Experience Pengguna).

Dengan berbagai tugas yang saya kerjakan pada minggu ketiga ini, saya bertekad untuk terus belajar untuk dapat menyelesaikan project yang telah diberikan dengan sebaik-baiknya. Kolaborasi dengan mentor sangat berharga bagi saya, karena memberikan arahan dan masukan yang dapat memperluas pemahaman saya dalam pengembangan perangkat lunak. Saya optimis ilmu yang telah diberikan oleh mentor dapat sangat bermanfaat bagi saya kedepannya.
